# conformance/

Cross-language golden vectors that pin the **composition ("glue")** around standard
crypto primitives — the layer where a polyglot (Rust + Go) deployment silently diverges
even when each language's primitives are individually correct.

`vectors.json` pins:

| Recipe | Standard | What it locks |
|---|---|---|
| `session_key` | RFC 5869 (+ FIPS 202 SHA3) | HKDF params: empty-salt vs 0-salt, `info` encoding |
| `frame` | NIST SP 800-38D | AES-256-GCM framing: 12-byte nonce, 16-byte tag appended |
| `fingerprint` | FIPS 180-4 | SHA-384 digest encoding |
| `signing_preimage` | RFC 8259 / ECMA-404 | canonical JSON: key order, minimal separators |

Both the Rust (`crates/forge-crypto`) and Go (`go/crypto`) providers assert **byte-equality**
against this file, so the ML-DSA signing preimage is provably identical across languages.

Primitives themselves (ML-DSA / ML-KEM) are KAT-locked to NIST FIPS 204 / 203 and are not
re-pinned here.

## Provenance

Generated by the `crypto-golden-vectors` (cgv) tool, which self-verifies each recipe at
generation (determinism, AEAD decrypt-back, canonical idempotence). No secrets; generic.
Apache-2.0.
